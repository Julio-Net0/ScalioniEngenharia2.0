03

Domínio vs Backend

O Princípio da Independência

Aplicamos o Dependency Inversion Principle para garantir que as regras de negócio nunca dependam de frameworks.

  • Domínio: Regras de negócio puras (sem FastAPI/SQLAlchemy).
  • Backend: Interfaces, Roteadores e Infraestrutura.

Vantagem Crucial

"Se amanhã trocarmos o FastAPI pelo Django, ou o PostgreSQL pelo MongoDB, o coração do negócio (Domínio) permanece intocado."

05

Especificações de Infra

O ecossistema é modernizado via Docker para garantir consistência entre ambientes.

  • Backend: Python 3.12 + FastAPI (Uvicorn).
  • Banco: PostgreSQL 16 (Async via SQLAlchemy 2.0).
  • Integração: Mercado Pago (SDK Oficial + Webhooks).
  • Migrações: Alembic (Controle de Versão de Schema).
  • Email: SMTP Seguro com e-mails transacionais.
  • Volume: Armazenamento persistente para plantas (.zip).
06

Fluxo de Webhook Seguro

Blindagem HMAC

Cada requisição do Mercado Pago é validada com um hash HMAC-SHA256, impedindo requisições falsas.

Double-Check

O sistema não confia apenas no Webhook; ele faz uma consulta ativa à API do MP para confirmar o status antes de liberar o download.

07

Autenticação e Senhas

Segurança Admin

O acesso ao painel administrativo é protegido por uma cadeia de confiança moderna.

  • JWT (JSON Web Token): Algoritmo HS256 para sessões stateless.
  • Bcrypt: Hashing de senhas com sal (Salt) dinâmico e custo computacional adaptativo.
  • Injeção de Dependências: Rotas protegidas via OAuth2PasswordBearer.

Anti-Brute Force

Implementamos o SlowAPI para limitar o número de tentativas de login por IP, mitigando ataques de força bruta.

08

Estratégia de Testes

A Pirâmide de Qualidade

Nossa suíte de testes não apenas mede cobertura, mas garante consistência real com banco de dados.

  • Pytest + Pytest-Asyncio: Framework base assíncrono.
  • Banco Real: Usamos containers PostgreSQL reais para os testes, validando Constraints e Foreign Keys.
  • Pytest-Cov: Monitoramento contínuo de cobertura.
09

O Desafio dos 80%

O projeto mantém uma média global acima de 80%, mas justifica as exceções pontuais:

Por que não 100%?

Webhook do Mercado Pago exige assinaturas HMAC complexas; priorizamos validação manual via Sandbox para maior realismo.

Foco no Domínio

Garantimos 100% de cobertura no Domínio. Blocos de erro defensivos na Infra (ex: servidor de e-mail fora) são raramente atingidos em CI mas fundamentais para resiliência.
